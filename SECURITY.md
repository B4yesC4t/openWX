# Security Policy

## Supported Versions

openWX is currently in the `0.x` phase. Security fixes are applied on the latest `main` branch first and may be backported only when maintainers decide the risk warrants it.

## Reporting a Vulnerability

Please do not open public GitHub issues for security reports.

Use a private disclosure path:

1. Prefer GitHub Private Vulnerability Reporting for this repository if it is enabled:
   <https://github.com/B4yesC4t/openWX/security/advisories/new>
2. If that flow is unavailable to you, contact the repository owner privately through:
   <https://github.com/B4yesC4t>

Include:

- A clear description of the vulnerability
- Affected package(s) and version(s)
- Reproduction steps or a proof of concept
- Any known impact or exploitability notes
- Whether the issue involves token leakage, message routing, media encryption, or persistence

## Response Process

- We will acknowledge receipt as quickly as practical
- We will validate the report privately before any public discussion
- We will coordinate a fix and release notes when remediation is ready
- We will credit reporters on request when disclosure becomes public

## Disclosure Expectations

- Give maintainers reasonable time to investigate and patch
- Avoid publishing exploit details before a fix or mitigation is available
- Use private channels for follow-up until maintainers confirm public disclosure is safe
